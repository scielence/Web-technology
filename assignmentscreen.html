6.) Look at the following two HTML paragraphs:

   
      			
      	<p>This is an <strong>important</strong> point.</p>

      	<p>This is another <b>important</b> point.</p>

Explain why the HTML of the first paragraph should be preferred. Illustrate this with an example. Also, identify another pair of HTML constructs for which the same rationale holds.

<b> just makes the text bold and is a style given to the text to make it appear more 'strong'. Whereas <strong> is an indication for user agents that this is truly an important text. For example, with browsers that assist blind people, <strong> would be used to emphasize on a spoken word and give it meaning, whereas <b> would just be rendered as normal.

8.) Describe the pros and cons of using frames in HTML.

One should be using frames because it allows for more content to be displayed at once and adds organization through layout and a variety of design options. Also allows for dynamic content.

One should not use frames because it causes for some user agents to possibly not display frames that are supposed to be displayed. Also, when (for ex.) packages go corrupt or get delayed, content could possibly go missing. Frames and framesets are also prone to not getting properly indexed by search engines.
